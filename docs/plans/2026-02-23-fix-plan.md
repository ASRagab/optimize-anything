## Task 2 — LLM Error Handling with Retry (C2)

**Depends on**: Task 1

**Issue**: `opts.model.generate(prompt)` at line 169 has no try-catch. Single API failure kills entire run.

**Files to modify**:
- `src/core/optimizer.ts` — wrap LLM call in retry logic

**Changes**:
1. Add helper `async function retryGenerate(model, prompt, maxRetries = 3): Promise<string | null>` at top of file
2. Uses exponential backoff: 1s, 2s, 4s delays between retries
3. On success: return response string
4. On all retries exhausted: return `null`
5. In main loop: replace bare `opts.model.generate(prompt)` with `retryGenerate()`
6. If `retryGenerate()` returns `null`: emit `{ type: "error", ... data: { reason: "llm_failure" } }`, `continue` to next iteration (skip this candidate)

- `tests/optimizer.test.ts` — "emits error event when all LLM retries exhausted": create fake LLM that always throws. Verify error event in result.events and optimization still completes gracefully.
